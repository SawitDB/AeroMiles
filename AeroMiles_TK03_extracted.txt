CRU – REGISTRASI (SEMUA) 

Sistem menampilkan halaman pilihan Login atau Register. Jika
 pengguna memilih Register, tersedia pilihan dua role: Member
 atau Staf. 

Skenario Registrasi Member: 

P engguna mengisi form: Email, Password, Konfirmasi Password,
 Salutation, Nama Depan-Tengah, Nama Belakang, Country Code,
 Nomor HP, Tanggal Lahir, Kewarganegaraan . 

Setelah mendaftar, akun Member akan otomatis masuk ke tier
 terendah (misal: Blue) dan mendapatkan nomor member yang
 di- generate sistem. 

C ontoh Implementasi: 

Skenario Registrasi Staf: 

P engguna mengisi form: Email, Password, Konfirmasi Password,
 Salutation, Nama Depan-Tengah, Nama Belakang, Country Code,
 Nomor HP, Tanggal Lahir, Kewarganegaraan, Kode Maskapai
 ( dropdown ). 

ID Staf di-generate otomatis oleh sistem. 

C ontoh Implementasi: 

Validasi : 

J ika ada field yang kosong atau tidak valid, sistem menampilkan pesan error . 

Jika email sudah terdaftar, sistem menampilkan pesan error
 duplikasi. 

Password dan konfirmasi password harus sama. 

Jika semua input valid, sistem membuat akun baru dan
 mengarahkan pengguna ke halaman login. 

C – LOGIN DAN LOGOUT (SEMUA) 

Fitur login dan logout dapat dilakukan oleh semua pengguna yang
 telah terdaftar pada sistem. 

Skenario Login: 

Sistem menampilkan halaman Login dan Register. 

Pengguna memasukkan email dan password lalu klik tombol Log
 In. 

Jika email atau password salah, tampilkan pesan error. 

Jika benar, session pengguna disimpan dan pengguna di- redirect ke Dashboard sesuai perannya. 

C ontoh Implementasi: 

Skenario Logout: 

Ketika pengguna telah berhasil masuk ke sistem dan ingin
 keluar dari sistem, maka pengguna akan menekan tombol 'Logout'
 yang terdapat pada Navbar. 

Dengan demikian, session pengguna akan berakhir dan akan
 kembali ke halaman "Login dan Register".

RU – PENGATURAN PROFIL (SEMUA) 

Fitur ini digunakan oleh pengguna untuk melihat dan memperbarui
 profil mereka 

Skenario Update Profil: 

Pengguna mengakses menu Pengaturan Profil dari Navbar. 

Sistem menampilkan form berisi data profil yang sudah
 terisi. 

Pengguna dapat mengubah: Salutation, First_Mid_Name,
 Last_Name, Country Code, Nomor HP, Kewarganegaraan, Tanggal
 Lahir. 

Email tidak dapat diubah. 

Khusus Member: Nomor Member dan Tanggal Bergabung tidak dapat
 diubah. 

Khusus Staf: ID Staf tidak dapat diubah. Kode Maskapai dapat
 diperbarui. 

Setelah menekan tombol Simpan, sistem memvalidasi dan
 menyimpan perubahan jika valid. 

C ontoh Implementasi: 

Skenario Ubah Password: 

Pengguna menekan tombol Ubah Password pada halaman Pengaturan
 Profil. 

Sistem menampilkan form: Password Lama, Password Baru,
 Konfirmasi Password Baru. 

Sistem memvalidasi password lama sebelum menyimpan password
 baru. 

C ontoh Implementasi:
